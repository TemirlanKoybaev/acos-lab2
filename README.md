# ACOS Lab 2 — TCP клиент–сервер, чат и история

Лабораторная работа по курсу «Архитектура компьютера и ОС».  
Цель — реализовать взаимодействие двух процессов с использованием TCP-сокетов

В работе реализованы:
- сервер, принимающий одного клиента за раз;
- простейший текстовый чат (произвольные сообщения);
- протокол ping–pong;
- команда `/history` для просмотра всей переписки;
- сохранение истории в файл `history.txt`;
- сборка с помощью CMake.

Проект написан на чистом C++17 с использованием POSIX-сокетов (`socket`, `bind`, `listen`, `accept`, `connect`, `send`, `recv`).

## Поддерживаемые команды

```
ping       → сервер отвечает "pong"
/history   → получить весь текстовый лог сообщений
любая строка → сервер отвечает "echo: <сообщение>"
quit       → выход из клиента
```

## Описание протокола

1. Клиент подключается к `127.0.0.1:5000`.
2. Каждое сообщение клиента записывается в `history.txt` с префиксом `CLIENT:`.
3. Сервер формирует ответ и также логирует его с префиксом `SERVER:`.
4. Для команды `/history` сервер отправляет весь файл истории клиенту.
5. Соединение закрывается клиентом или при обрыве связи.
